# レシピ帳 — 実装ハンドオフ仕様

自分専用のレシピ管理 PWA。クラフト紙のような生成り地＋テラコッタ差し色、明朝×ゴシック、静かな道具のようなトーン。モバイル縦・片手操作主軸。参照プロトタイプ: `レシピ帳.dc.html`（同梱。挙動・レイアウト・コピーの正本）。

## 1. プロダクト原則
- 自動取り込みは便利だが**主役ではない**。手入力体験を邪魔しない控えめな存在感。
- 一覧は**写真主役ではなくテキスト主役**（ユーザーは完成写真を載せない運用。画像はレシピのスクショを自由記載欄に貼る）。
- 料理中に片手で読めるサイズ・コントラスト。主要ボタン（保存・作った・FAB）は親指の届く下部に。
- 全データは端末内。ライト基調（ダークは将来対応で可）。

## 2. 画面一覧（実装順）
1. **ホーム（データベース）**: 検索バー／タグ絞り込みチップ（横スクロール・複数選択＝OR）／ソート切替（古い順・高評価順・最近追加順）／レシピ一覧／新規追加 FAB。一覧は単列のテキストカードが既定。空状態あり。
2. **レシピ詳細・編集**: 画像ギャラリー（横スクロール）／タグ／レシピ名（明朝大）／星評価（タップで設定）／URL リンク／最終作成日／本文（Markdown 表示）。下部に目立つ「**作った**」ボタン＋「**リスト**（に追加）」ボタン。編集モードで各フィールドを入力化。編集モード上部に **URL 貼り付け取り込みバー**（§5）。
3. **リスト一覧**: 手動リスト（実サムネ＝色バー）とスマートリスト（破線・「自動」バッジ）を視覚的に区別。各リストの件数表示。新規リスト作成（手動/スマートを選ぶシート）。
4. **リスト詳細**: 手動＝ドラッグで並べ替えできるカード一覧＋「レシピを追加」（レシピ選択シート）。スマート＝条件サマリ＋「条件を編集」＋抽出結果一覧。
5. **スマートリスト条件設定**: タグ複数選択＋「すべて含む/いずれか含む」切替／最低星評価（0=指定なし）／最終作成日（指定なし・2週間・1ヶ月・2ヶ月・3ヶ月以上＝N日以上未作成）／キーワード／合致件数のライブ表示／名前を付けて保存。
6. **設定**: バックアップ（エクスポート/インポート・最終バックアップ日時）／データ件数／ヘルプ「レシピ取り込みショートカット」（→ 手順シート §5.3）。
7. **取り込み画面（共有/貼り付け）**: paste → loading → review の3段。§5.1。

## 3. データモデル（types.ts）
```ts
type RecipeImage = {
  type: 'url' | 'local';     // url=元サイトから取り込み（リンク切れ得る）/ local=自前写真
  src: string;               // url: 画像URL / local: 端末内参照(blob/IndexedDBキー)。プロトは色で代用
  broken?: boolean;          // url画像が読み込めない時 true → プレースホルダ表示
};

type Recipe = {
  id: string;
  name: string;
  rating: number;            // 0–5（タップで設定）
  lastMade: string;          // ISO日付 'YYYY-MM-DD'
  added: number;             // 追加順（recent ソート用、新規は最大）
  tags: string[];
  url: string;               // 参考レシピURL（任意）
  body: string;              // Markdown（材料・手順・メモ。スクショもここに）
  images: RecipeImage[];
};

type ManualList = { id: string; name: string; type: 'manual'; recipeIds: string[] }; // 並び順=配列順

type SmartCondition = {
  tags: string[];
  tagMode: 'all' | 'any';    // すべて含む / いずれか含む
  minRating: number;         // 0=指定なし。recipe.rating >= minRating
  notMadeDays: number;       // 0=指定なし。daysSince(lastMade) >= notMadeDays
  keyword: string;           // recipe.name に含む（本文も対象にするかは要相談、プロトは名前のみ）
};
type SmartList = { id: string; name: string; type: 'smart'; cond: SmartCondition };
```
**スマート抽出ロジック**（プロト `evalSmart` 準拠）: 全件に対し keyword一致 ∧ rating≥minRating ∧ (notMadeDays>0 なら経過日数≥notMadeDays) ∧ タグ条件（all=全タグ保持 / any=いずれか保持）を満たすものを返す。

シードデータ・初期リスト・スマートリスト2件はプロトの `constructor` 内 `recipes/manualLists/smartLists` をそのまま移植。

## 4. 主要インタラクション
- **星評価**: 詳細で星をタップ→その値に設定。
- **「作った」ボタン**: タップ一発で `lastMade=今日`、軽いポップ＋トースト「『◯◯』を作りました」。記録のしやすさが肝。
- **リストに追加**: ボトムシートで手動リストをチェックトグル。
- **タグ入力**: 既存タグのサジェスト（入力で絞り込み、タップで追加、×で削除）。
- **手動リスト並べ替え**: ドラッグ＆ドロップで `recipeIds` を入れ替え永続化。タッチ対応必須（プロトは HTML5 drag で簡易実装＝モバイルは要 pointer ベース実装に置換）。
- **検索/絞り込み/ソート**: ホームでリアルタイム反映。

## 5. 取り込み機能
### 5.1 取り込み画面（FAB→「URLから取り込む」/ 共有経由）
- **paste**: ペースト促進プレースホルダ＋ワンタップ「クリップボードから貼り付け」(`navigator.clipboard.readText()`)→「取り込む」（URL 空は無効）。
- **loading**: スピナー＋「ページを読み込んでいます」＋対象URL。
- **review（プリフィル状態）**: やわらかいバナー「自動で取り込みました。内容を確認して保存してください。」＋サムネ右上に控えめな「サイトから取得」リンクバッジ。名前/URL/説明を編集可、目立つ「保存する」。保存で `images:[{type:'url',...}]` の Recipe を作成。
- iOS ショートカット共有経由は **review に直接入る**だけで UI は同一。
### 5.2 編集画面のURL貼り付けバー
詳細の編集モード上部。URL貼付＋ペースト＋「取り込む」→インラインのスピナー→現レシピの name/url/body/画像へ反映。
### 5.3 ショートカット案内
設定のヘルプ項目→手順シート（3ステップ）＋「ショートカットを追加」＋「取り込み画面をプレビュー」。
### 5.4 実装注意（URLメタ取得）
- まず `fetchRecipeMeta(url)` を**モック**で（プロト `fakeFetch` 相当：名前・説明・画像色を返す）。
- 実運用は OGP/JSON-LD(recipe schema) 取得が必要。ブラウザ直 fetch は CORS で不可 → **軽量プロキシ or iOS ショートカット側で取得**して渡す設計に。ここは設計判断が要るので実装前に相談。
### 5.5 画像の出自表示
- `type:'url'` → サムネ右上に小さなリンクアイコンバッジ。`type:'local'` → 無印。
- `broken:true`（url読込失敗）→「画像を読み込めません／元サイトのリンク切れ」プレースホルダ。`<img onError>` で broken を立てる。

## 6. デザイントークン
- 地: `#F4EDE0`（アプリ背景）/ サーフェス `#FBF6EC` / 罫線 `#E6DAC6`
- インク: `#2E2620`（本文）/ サブ `#6B5F52` / 補助 `#9A8C7A`
- 差し色（テラコッタ）: `#B6492F` / 淡 `#F2E0D7` `#F6EADB`
- 星: 点灯 `#B6492F` / 消灯 `#DBCDB6`
- フォント: 見出し・レシピ名＝**Shippori Mincho**（明朝）/ UI・本文＝**Zen Kaku Gothic New**（ゴシック）
- 角丸 13–18px、影は薄く（`0 1px 2px rgba(60,46,30,.06)` 程度）。余白広め。
- 1920換算の話ではなくモバイル: 本文14px以上、ボタンタップ領域44px以上。
- アクセシビリティ: 料理中向けに十分なコントラストと文字サイズ。

## 7. PWA要件
- `manifest.json`（アイコン・テーマ色 `#F4EDE0` / `#B6492F`・standalone・縦固定）
- Service Worker でオフライン（アプリシェル＋データは IndexedDB）
- バックアップ＝全データの JSON エクスポート/インポート（設定画面。最終バックアップ日時を保存）
